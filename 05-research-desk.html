Multi-Agent Research Desk

Multiple agents talk to each other and work autonomously. Stand up a small team of agents that work together: a collector, a writer, and a reviewer hand a job through a shared store to produce a brief — escalating anything they're unsure about to a human. A deployed board shows each job moving through the stages.

Why this assignment

The capstone. One prompt doing everything is easy; several agents cooperating through clean handoffs, escalating when unsure, and recovering from failure is the real test. We want to see your boundaries: what each agent owns, how they pass work, and where a human steps in.

Use any AI coding tools. We grade architecture and judgement — the handoff contract and the escalation logic — not the number of agents. Aim for production-grade — clean, reliable, something a teammate could actually run.

The brief

  1. OpenClaw or Hermes on a free-tier VPS running ≥3 cooperating agents.
  2. Submit a research job (a topic/question). A Collector gathers sources, a Writer drafts a brief, a Reviewer checks it.
  3. Agents hand off via a shared store (a database table is the queue) with a clear handoff record — not one prompt doing everything. The Reviewer escalates low-confidence briefs to a human-review lane instead of publishing. Failures retry.
  4. A deployed frontend is a live board: jobs move queued → collecting → writing → review → published / escalated, with the artifact at each stage.

Live, public URL, running ≥1 week unattended.

Example — what a great submission looks like: we open your board, submit "State of on-device LLMs 2026," and watch a card move queued → collecting → writing → review. The Collector logs ~5 sources, the Writer drafts a brief, the Reviewer flags low confidence → the card lands in "Needs human" instead of publishing. We kill the Writer mid-run and it retries. Three distinct agents, each owning a stage, passing a JSON handoff record through the shared store. (Going further: a durable workflow like Temporal, idempotent handoffs, pipeline observability.)

What you hand over

  2. A public GitHub repo URL — code, README, commit history.

Vendors are your choice. Stuck on infra or free credits? Ask us — figuring out what to build is the test; we'll happily help with resources.

LLM provider: the agents need one. Ollama Cloud free tier (rate-limited — design assuming it will throttle), or any LLM key you already have. Pick any job domain you like (e.g. "research a topic → draft → review/publish", or "triage an issue → propose a fix → review").

Ground rules

  1. Free tiers only. Few agents, small models, sparing runs — cap tokens.
  2. Real handoffs, not one mega-prompt. Each agent owns a stage and passes a documented record to the next — graded directly.
  3. Escalate, don't auto-ship. Low-confidence results go to a human lane; failures retry, not crash.
  4. Keep it running ≥7 days. README must let us redeploy from scratch.
  5. Blocked? Tell us. We'd rather unblock you on infra/credits than have you stuck — just ask.
